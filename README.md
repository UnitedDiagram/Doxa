# Doxa

AI-powered equity research pipeline. Give it a stock ticker and it runs a
six-agent analysis pipeline — market data, valuation, regulatory filings,
sentiment, report writing, and editorial distillation — to produce a
high-signal equity research report, viewable in a Streamlit web UI or
exportable as a PDF.

## How it works

All agents read and enrich a shared `ResearchState` dict, in sequence:

```
Ticker
  │
  ▼
MarketDataAgent   — price history & fundamentals via yfinance
  ▼
ValuationAgent    — DCF model, comps, DuPont analysis, Altman Z-Score, ratios
  ▼
RegulatoryAgent   — SEC EDGAR 10-K risk analysis via Claude
  ▼
SentimentAgent    — insider trading, short interest & sentiment signals via Claude
  ▼
WriterAgent       — comprehensive Markdown research report via Claude
  ▼
EditorAgent       — distills to high-signal content (non-obvious risks,
  │                 thesis drift, mispriced assumptions)
  ▼
Report (Streamlit UI / Markdown / PDF)
```

The pipeline degrades gracefully: data-fetch or API failures are recorded in
`state["errors"]` and the run completes with partial data rather than crashing.

## Requirements

- Python 3.12+
- An [Anthropic API key](https://console.anthropic.com/) for the
  Claude-powered agents
- For PDF export (optional): WeasyPrint's native libraries
  (`brew install pango` on macOS, `apt-get install libpango-1.0-0
  libpangoft2-1.0-0` on Debian/Ubuntu)

## Quickstart

```bash
git clone <this-repo> && cd doxa
python -m venv .venv && source .venv/bin/activate
pip install -e ./shared
pip install -e ".[dev]"

cp .env.example .env   # then add your ANTHROPIC_API_KEY

streamlit run app.py   # web UI
python -m src.main     # CLI
```

## Configuration

Set via environment variables or a `.env` file in the project root:

| Variable | Default | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | (required) | Claude API key |
| `DOXA_LOG_LEVEL` | `INFO` | Logging level |
| `DOXA_HISTORY_PERIOD` | `1y` | yfinance price history window |
| `DOXA_MAX_HEADLINES` | `10` | Max headlines for sentiment analysis |

## Notes & limitations

- **Alternative data is mocked.** The SentimentAgent's insider-trading /
  short-interest / options-flow feed (`doxa_shared.utils.sentiment_data`)
  returns structured mock data. Wiring in a real provider (e.g. Quiver
  Quantitative, FinBrain) is left to you.
- **SEC EDGAR User-Agent.** The SEC requires API consumers to identify
  themselves. Before heavy use, replace the placeholder contact in
  `shared/src/doxa_shared/utils/edgar.py` with your own.
- **yfinance is unstable.** Field names and data structures change between
  versions; the code parses defensively, but missing data is normal.

## Project layout

